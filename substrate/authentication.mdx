# Authentication

The Substrate API uses API keys for authentication. All API requests must include your API key in the request headers.

## Getting Your API Key

1. Sign up for a Substrate account at [https://dashboard.substrate.ai](https://dashboard.substrate.ai)
2. Navigate to the API Keys section in your dashboard
3. Generate a new API key for your application
4. Store your API key securely

<Warning>
  Keep your API key secure and never expose it in client-side code. API keys should only be used in server-side applications.
</Warning>

## Authentication Methods

### Bearer Token (Recommended)

Include your API key in the `Authorization` header using the Bearer token format:

```bash
curl -X POST https://api.substrate.ai/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-4",
    "messages": [
      {
        "role": "user",
        "content": "Hello, world!"
      }
    ]
  }'
```

### API Key Header

Alternatively, you can include your API key using the `X-API-Key` header:

```bash
curl -X POST https://api.substrate.ai/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{
    "model": "gpt-4",
    "messages": [
      {
        "role": "user",
        "content": "Hello, world!"
      }
    ]
  }'
```

## Code Examples

<CodeGroup>
```python Python
import requests

api_key = "your-api-key-here"
base_url = "https://api.substrate.ai/v1"

headers = {
    "Content-Type": "application/json",
    "Authorization": f"Bearer {api_key}"
}

response = requests.post(
    f"{base_url}/chat/completions",
    headers=headers,
    json={
        "model": "gpt-4",
        "messages": [
            {"role": "user", "content": "Hello, world!"}
        ]
    }
)

print(response.json())
```

```javascript JavaScript
const apiKey = 'your-api-key-here';
const baseURL = 'https://api.substrate.ai/v1';

const response = await fetch(`${baseURL}/chat/completions`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${apiKey}`
  },
  body: JSON.stringify({
    model: 'gpt-4',
    messages: [
      { role: 'user', content: 'Hello, world!' }
    ]
  })
});

const data = await response.json();
console.log(data);
```

## Error Handling

If authentication fails, the API will return a `401 Unauthorized` status code:

```json
{
  "error": {
    "type": "authentication_error",
    "message": "Invalid API key provided",
    "code": "invalid_api_key"
  }
}
```

## Rate Limits

API keys are subject to rate limits based on your subscription tier:

- **Free Tier**: 1,000 requests per day, 10 requests per minute
- **Pro Tier**: 100,000 requests per day, 500 requests per minute
- **Enterprise**: Custom limits available

Rate limit headers are included in all API responses:

```
X-RateLimit-Limit: 500
X-RateLimit-Remaining: 499
X-RateLimit-Reset: 1640995200
```

## Best Practices

<AccordionGroup>
  <Accordion title="Secure Storage">
    - Store API keys in environment variables
    - Use a secure secret management system in production
    - Never commit API keys to version control
    - Rotate API keys regularly
  </Accordion>

  <Accordion title="Error Handling">
    - Always check for authentication errors
    - Implement proper retry logic with exponential backoff
    - Log authentication errors for monitoring
  </Accordion>

  <Accordion title="Rate Limiting">
    - Monitor your rate limit usage
    - Implement client-side rate limiting
    - Cache responses when appropriate
    - Use batch requests when available
  </Accordion>
</AccordionGroup>
