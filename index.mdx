---

#### How to Use

- You **must specify both** `platform` and `model` in every request.
- If the model is not available for a specified platform, you'll get an `INVALID_MODEL` error.
- Live model/platform additions are updated in the Substrate dashboard, not always in static docs.

#### Example API Calls

**OpenAI:**
```json
{
  "messages": [{ "role": "user", "content": "Summarize this text" }],
  "platform": "openai",
  "model": "gpt-4o"
}
```

#### Discovering New Models & Platforms

- **Best practice:** Always check your [Substrate dashboard](https://app.getsubstrate.co) for the newest models and supported platforms.
- **Programmatic access:** If available, query `/models` or similar endpoint for current options.

---

#### Troubleshooting

- **`INVALID_MODEL` error:** Usually means the model string is misspelled, not active for that platform, or out of date.
- **Action:** Double-check your spelling, permitted models, and that you're pairing platform/model correctly as above.
